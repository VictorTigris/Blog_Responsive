- ¿Qué es este Blog? -

Es lógico que te hagas esta pregunta, ya que en un principio, nada de este sitio web parece tener una verdadera funcionalidad.
Sin embargo, sí tiene una... literalmente, solo tiene una. Y es la de ayudarme a mí mismo a seguir todos mis propios avances en la creación de este sitio web. Es algo así como una meta-página que habla de sí misma, de su nacimiento y mantenimiento. He decidido comenzar a publicar esto por dos motivos; por un lado me sirve de excusa para tener algo que crear; y por el otro para crear algo desde cero, cien por cien responsive design.

¿Qué pasa con GitHub, GitLab, etc..?

Lo sé, existen múltiples herramientas ideales para el seguimiento de avances, o mejor llamado "versiones". De hecho yo mismo utilizo Git junto con GitHub, puedes ir a ver mis repositorios cuando quieras desde el menú de este sitio.

Entonces, ¿qué busco con este blog?

Un seguimiento personal. Con GitHub puedo tener un control absoluto de mi código y mis versiones, pero no describir de manera holgada y relajada mis experiencias. Se trata de ser preciso en GitHub, y no queda lugar para los comentarios, los consejos, los recordatorios.
Es por esto que además de ir actualizando mi repositorio remoto voy a ir posteando mis avances en este blog.
Mi método será el de postear mis tags de GitHub tal cual en la página principal de este sitio web. De ese modo se podrá ir viendo siempre la versión más reciente en la parte superior.
Cada una de esas etiquetas de GitHub podrá llevarte al commit original del repositorio remoto, y también a la parte del blog en la que hable de ese avance en concreto.

Espero que, si decides explorar algo de este blog, te sirva para aprender al menos un poco, ya sea de mis aciertos o de mis errores.
